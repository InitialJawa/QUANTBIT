# LAPORAN AUDIT DATA SINTETIS — QUANTBIT

**Tanggal Audit**: 16 Juni 2026
**Auditor**: AI Static Code Analysis
**Keyakinan**: ~95% — **Project menggunakan data sintetis secara signifikan**

---

## RINGKASAN EKSEKUTIF

Project QUANTBIT tidak menggunakan library pembangkit data sintetis khusus (Faker, CTGAN, SDV, SMOTE, Mimesis, dll.), tetapi membangun data sintetis secara **manual** melalui 5 teknik:

| Teknik | Deskripsi |
|--------|-----------|
| **Procedural Generation** | Hash deterministik dari string ticker untuk menghasilkan angka pseudo-random |
| **Math.random()** | Random walk harga saham di UI |
| **Carry-Forward** | Data hari sebelumnya diisi ulang saat Yahoo Finance gagal |
| **Hardcoded Static** | Data statis yang diklaim sebagai data live/aktif |
| **Mock/Simulasi** | Portfolio palsu dengan flag `_simulated: true` |

## DATA STATUS ENUM (SUDAH ADA)

`src/types/DataStatus.ts`:
- `LIVE` — real-time dari API
- `CACHED` — data tersimpan, masih valid
- `STALE` — data kedaluwarsa
- `ESTIMATED` — data generated by model/fallback algorithm

> **Catatan**: Enum sudah dibuat tetapi **belum diimplementasikan secara konsisten** di seluruh komponen.

## REKOMENDASI

### Segera (High Priority)
1. Hentikan `Math.random()` untuk harga di UI (`src/App.tsx:218`)
2. Terapkan `DataStatus` secara konsisten ke semua komponen yang menampilkan data finansial
3. Labeli semua data statik (`marketData.ts`, `live_market.json`) dengan status `CACHED` / `STALE`
4. Jangan overwrite cache valid dengan array kosong (`data/idx80_scan.json`)

### Jangka Pendek
5. Ganti chart generated (`seedFromTicker`) dengan real data atau label `ESTIMATED`
6. Ganti fundamental fallback sintetis dengan "Data Tidak Tersedia"
7. Beri label `CARRIED_FORWARD` pada data hasil bridging (`bridgeHistoricalDataToToday`)
8. Audit carry-forward di `fetch_historical_data.ts` — bedakan data asli vs carried

### Jangka Panjang
9. Pisahkan portfolio simulasi dari portfolio user asli di UI
10. Implementasikan data freshness guard berdasarkan timestamp asli, bukan timestamp generasi
