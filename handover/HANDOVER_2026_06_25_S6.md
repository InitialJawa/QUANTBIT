## What Shipped

### Level 1 — Smarter Q&A
- **History persist**: `localStorage.setItem("quantbit_ai_chat_history", ...)` on every messages change, cap 100 pesan.
- **Welcome message**: prepended kalau history kosong.
- **Trash button** di header chat untuk clear history (`setMessages([WELCOME])` + `localStorage.removeItem`).
- **Richer live context**: `buildLiveContext()` sekarang kirim:
  - `bps` (Buy Pressure Score) — score, action, deployPct, valid, reason, 5 factors
  - `backtestConfigSnapshot` — profile, mode, universe, topN, crash, dcaActive
  - `isBacktestOutOfSync` — true kalau backtestConfig diverges dari engineConfig
  - `alerts` (last 5) — recent fired notifications
- **AILiveContext** type extended di `systemKnowledge.ts` (backward compat — semua field optional).

### Level 2 — Read-only Tool Use (8 tools)
- **Mekanisme**: model emit JSON block `{"tool_call": {"name": "...", "args": {...}}}` di response text. Frontend `extractToolCalls()` regex-parse.
- **Backend**: tidak perlu perubahan (system knowledge Section 13 instructs model untuk emit JSON blocks; provider chain unchanged).
- **Tool registry** di `src/hooks/useAITools.ts`:
  - `get_portfolio_state()` — positions, cash, watchlist
  - `get_bps_now({ticker?})` — current BPS (market-level; per-ticker not implemented)
  - `get_regime_details()` — regime status, breadth, drawdown, ihsgTrend
  - `get_ticker_metrics({ticker})` — current price, PE/PB/ROE/DER, rank, final score, profile
  - `get_market_history({days?})` — last N days IHSG
  - `get_backtest_config()` / `get_engine_config()` — current settings (separate)
  - `get_active_universe()` — tickers user cares about (custom mode)
- **Tool execution flow** di `FloatingAIChat.send()`:
  1. Send messages + ctx to backend
  2. Receive `{ content, toolCalls }`
  3. For each read-only tool call: `executeTool(tc)` → push result as "tool" message
  4. For each action call: `addPendingAction(buildPendingAction(action))` → render card
  5. If any tool executed: re-ask AI with tool results appended (follow-up turn)

### Level 3 — Action API + Inline Card (10 actions)
- **Action registry** di `useAITools.ts` (mirror of read-only registry, but returns `AIAction`):
  - `buy_stock({ticker, shares, price?})` → dispatch ke `pm.handleAddTransaction`
  - `sell_stock({ticker, shares})` → `pm.handleSellTransaction`
  - `move_to_gold({rupiahAmount})` → `pm.handleMoveToGold`
  - `set_active_profile({profileId})` → `setActiveProfile`
  - `set_universe({universe})` → `updateConfigValue("universe", ...)`
  - `set_topN({n})` → `updateConfigValue("topNCount", ...)`
  - `toggle_dca_active({active})` → `updateConfigValue("dcaActive", ...)`
  - `add_to_watchlist` / `remove_from_watchlist` → `pm.handleToggleWatchlist` (with state check)
  - `sync_backtest_to_portfolio` → `syncFromBacktest(snapshot)` dengan snapshot di-rebuild manual
- **`AIActionApprovalCard`** component — inline card dengan:
  - Display text (e.g. "Beli 100 lembar BBCA @ Rp 10.500")
  - Impact preview list (e.g. "Estimasi biaya: Rp 1.05M", "Kas tersedia: Rp 100M", "⚠ Biaya melebihi kas")
  - [Approve] [Reject] buttons (Reject disabled when executing)
  - Status feedback: "✓ Executed" / "✗ Rejected" / "✗ {error}"
- **Approval flow**: `addPendingAction` → `AICockpitContext.pendingActions` queue → card render di bawah chat messages → user klik [Approve] → `executeAIAction(pending)` → `approveAction(id)` removes from queue.
- **Zero auto-execute**: AI SELALU propose, user SELALU confirm.

### Level 4 — Proactive Agent
- **`useProactiveAgent` hook** dengan 6 rules + 5-min hardcoded cooldown per rule:
  1. `bpsAggressive` — BPS 70-89
  2. `bpsDeploy` — BPS ≥ 90 (capitulasi)
  3. `bpsLow` — BPS < 30
  4. `dcaOffHighBps` — BPS ≥ 80 tapi `dcaActive=false`
  5. `crisisOverride` — `isCrisisMode()` aktif
  6. `ihsgDrop` — IHSG monthly drop > crashSensitivity
- **Cooldown**: `lastFiredRef.current[ruleId] = Date.now()` per rule. `canFire` check `now - last >= 5*60*1000`.
- **Toggle**: `proactiveAIEnabled` di `useUIState` (localStorage `idx_proactive_ai`, default true). Settings menu di AppHeader ada toggle row "Proactive Alerts" dengan Bell/BellOff icon.
- **Mount**: `<ProactiveAgentBridge />` di `App.tsx` (dalam AICockpitProvider + NotificationProvider).
- **Chat badge**: `FloatingAIChat` menampilkan amber Bell badge + unread counter di tombol chat.

## Why JSON-Block (Not Native Function Calling)

Backend spec di `AI_DEPTH_UPGRADE_PLAN.md` Section 5 suggests native function declarations (Gemini `function_declarations` + OpenAI `tools`). **Chose JSON-block approach** instead karena:
1. **Portable**: works dengan OpenRouter, Groq, dan Gemini tanpa 3 schema berbeda
2. **No backend changes**: backend tidak perlu maintain `functionCall` parsing untuk 3 providers
3. **Simpler**: model emit JSON inline, regex extract
4. **System knowledge Section 13 sudah specify format ini** sebagai primary contract

Trade-off: model harus lebih disciplined emit JSON block (kadang lupa wrap). Follow-up logic di frontend bisa handle kalau model emit plain text tanpa tool call.

## Known Limitations (Future Work)

- **Per-ticker BPS** not implemented — `get_bps_now({ticker})` returns global BPS + note. Would need per-ticker score in `useBuyPressure`.
- **Follow-up turn** is best-effort — if model emits new tool calls in followup, they're collected for actions but read-only tool results are not executed again.
- **`proactiveAlerts` queue in AICockpit** is wired but not yet rendered as a chat-anchored chip. Currently only `useNotifications` toast is shown.
- **No auto-execute** for BPS recommendation (Level 5) — explicitly out of scope per plan.
- **Chat history cap 100** — older messages dropped. No export.
