## BPS Scoring Design

```
Valuasi (30%) = avg stock value score (1/PE) dari leader universe
Momentum (25%) = clamp(50 - ihsgMonthly * 2, 0, 100)  // IHSG turun = high
Breadth  (15%) = (1 - breadth_above_60 / watchlist_count) * 100
Drawdown (20%) = min(100, -drawdown60 * 4)  // drawdown -25% = 100
Fear     (10%) = RS.risk langsung  // regime risk = panic indicator
```

**Action map** (per PRD):
- 0-30: tidak beli
- 30-50: beli kecil (25% kas)
- 50-70: beli normal (50% kas)
- 70-90: beli agresif (75% kas)
- 90-100: deploy hampir semua (90% kas)
- **Override**: `isCrisisMode()` → `valid: false, action: none` (CASH DEFENSE priority)

## Backtest Mode: `adaptive_dca`

For each day in the historical period:
1. Track 60-day rolling window of IHSG prices
2. Each month change: compute BPS from historical signals
   - `ihsgMonthly` = 21-day rolling return
   - `drawdown60` = peak-window drawdown
   - `breadthAbove60` = stocks with rank ≤ topN × 3
   - `riskScore` = `abs(ihsgMonthly) * 4` clamped 0-100
   - `avgValueScore` = avg of stock value scores
3. Deploy `BPS.deployPct` of available cash to Top N (equal weight via `computeInitialAllocation`)
4. Hold positions — no monthly rebalancing
5. Crash detection still works (exit to gold)

Returns `bpsHistory` (per-month BPS snapshot) + `totalDeployed` for analysis.

## 4-Way Comparison (SimulationTab)

When user runs backtest in `adaptive_dca` mode, after the main backtest completes, three baselines are computed in parallel:

| Strategy | Logic | Best for |
|----------|-------|----------|
| **Adaptive DCA** | BPS-driven deploy monthly | Bear/capitulation markets |
| **Lump Sum** | Buy Top N day 1, hold | Bull markets |
| **Monthly DCA** | Invest capital/12 every month | Steady accumulation |
| **Quarterly DCA** | Invest capital/4 every quarter | Less frequent deploy |

Each card shows: final value, return %, CAGR, Max DD, Avg Buy Price, Cash Utilization.
Verdict banner: "Adaptive DCA mengungguli [best baseline] sebesar X.X poin" (or underperform).

## Known Limitations

- **Fear Indicator** uses `RS.risk` (regime risk score) as proxy. Real VIX-like indicator not implemented (no VIX data in pipeline).
- **Momentum sub-score** in backtest uses 21-day rolling IHSG return, not the live `MKT.ihsg.monthly` (which is only available for current day).
- **No auto-execution** (Phase 3) — BPS is recommendation only. User must manually deploy.
- **Baseline crash protection disabled** for fair comparison (baselines are pure DCA, no defensive behavior).
- **Custom mode not adapted** — `custom` mode still uses topN rank-based exit. `adaptive_dca` is separate.
