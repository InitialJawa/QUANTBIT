## 3 Mode + 1 Bridge (User Mental Model)

```
┌─ Default: LIVE (toggle ON) ──────────────────────────┐
│ Backtest = engineConfig. Pasti koheren.              │
│ Strategy fields di sidebar greyed.                   │
│ Banner: ✓ Live Strategy                               │
└──────────────────────────────────────────────────────┘
            ↕ toggle "Gunakan Strategi Portofolio"
┌─ DRAFT (toggle OFF) ─────────────────────────────────┐
│ Backtest = backtestConfig sandbox.                   │
│ Strategy fields editable.                            │
│ Banner: ⚠ DRAFT — perubahan tidak effect Portofolio │
│                                                       │
│ ┌─ Bridge: "PROMOTE TO PORTFOLIO" button ──────────┐│
│ │ Click → engineConfig = backtestConfig          ││
│ │ → Auto-toggle to ON + toast success            ││
│ └─────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘

Edge case: Toggle ON dengan draft unsynced
  → ConfirmModal 3 opsi:
    • Buang, pakai Portofolio (discard draft)
    • Promote Dulu, Baru Aktifkan (apply draft first)
    • Batal (stay in DRAFT)
```

## Notes for AI agents picking up from this handover

- **CRITICAL**: `engineConfig` is the SINGLE SOURCE OF TRUTH for all strategy fields. Read it for live state.
- **Backtest**: when `backtestUseLiveStrategy=true` (default), the run config is `effectiveConfig = {...backtestConfig, ...engineConfig}` — engineConfig wins for 10 strategy keys.
- **StrategySettingsPanel**: only component that renders 10 strategy fields. Used identically in Portfolio + Backtest sidebar.
- **Adaptive Weights** is REMOVED from UI (was in Backtest sidebar previously, deprecated by ADR-010, now truly gone).
- **Settings Lock** + **DCA Rekomendasi** are Portfolio-only. They live in Portfolio sidebar section, NOT in StrategySettingsPanel.
- **isConfigSynced** deleted from context. Use `isDraftEqualToEngine()` method instead.
