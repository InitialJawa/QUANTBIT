### 2. DB Sync Status UI
- **useDataFeed.ts** — NEW: fetch `/api/db-sync-status`, expose `syncStatus` + `triggerSync()`
- **PortfolioTracker.tsx** — Sync status bar (last synced date, stale warning badge, "Sync Now" button)
- **MarketTab.tsx** — DB sync indicator chip in status bar
- **Visual**: Live prices tagged `[STALE]` when DB older than 1 day

### 3. Decision Engine DB SOT
- **src/marketRegimeEngine.ts** — 6 functions refactored:
  - `isCrisisMode()` — use DB last close (`closes[last]`), not Yahoo live (`MKT.ihsg.value`)
  - `getIhsgDrawdown60()`, `getIhsgMonthlyReturn()`, `getIhsgWeeklyReturn()`, `getIhsgDailyReturn()`, `computeMarketRegime()`
  - All use DB closes, ignore live prices for calculations
- **PortfolioTracker.tsx** — `evaluateStrategy()` + `rule_crashProtectionTriggered()` use DB close
- **Impact**: Portfolio & Backtest now agree on crash detection (previously diverged ~5-10%)

### 4. Crisis State Machine Hysteresis
- **Module-level state**: `_crisisInCrisis`, `_crisisCooldown`, `_crisisLastDataLen`, `_crisisInitialized`
- **Lazy init** (`_initCrisisState()`) — fast-forward state machine through ALL historical IHSG data
- **Cooldown tracking** — increments only when new data arrives (not per-render)
- **Reset triggers** — `setIhsgHistory()`, `setCrashSensitivity()`, `setCrashProtectionEnabled(false)`
- **Result**: Portfolio DCA state now matches backtest end state exactly (no false-positive crisis exits)

### 5. Bug Fixes (Session 14b)
- **Yahoo batchSize 50 → 20** — Spark endpoint max 20 symbols per request (was HTTP 400)
- **Gold price conversion** — Yahoo returns USD/oz, convert to IDR/gram (`÷31.1035 × USD/IDR`)
- **`--force` flag parsing** — Date vs flag separation in arg parser (was `Invalid time value`)
- **seed-db.py encoding** — Unicode `→` → ASCII `->` (cp1252 error on Windows)
- **PYTHONIOENCODING=utf-8** — Set on all Python execSync calls in server.ts
