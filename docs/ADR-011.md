# ADR-011: Unified Settings Panel + Backtest‚ÜîPortfolio Default Coherence

**Status:** Accepted
**Date:** 2026-06-26
**Deciders:** @InitialJawa

## Context

User melaporkan (Sesi 12):

> "backtest udah masuk fase pemulihan dan udah membeli saham lagi tapi pas di porto dan market malah masih crash. kalau kek gtu 2 mesin itu baca data yang beda atau ada setting yang ngga sync semua"

Dan:

> "settingan ui yang ada di porto dan backtest jujur beda jauah itu yang buat kita ngga bisa sych"

### Audit menemukan 2 masalah utama

**1. Sinkronisasi config bocor**
- `backtestConfig` (draft sandbox) dan `engineConfig` (live) bisa diverge tanpa warning yang jelas
- Sidebar menampilkan warning `‚öÝ Backtest config differs` tapi subtle dan user-skipable
- Hasil: backtest jalan dengan `backtestConfig` (mis. crashSensitivity=15), portfolio pakai `engineConfig` (crashSensitivity=10) ‚Üí sinyal bertentangan

**2. Settings UI sangat tidak konsisten**
- Portfolio sidebar: 200+ LOC inline (`EngineConfigSidebarContent`) dengan 10 strategy fields
- Backtest sidebar: 100+ LOC inline dengan order berbeda + Adaptive Weights (deprecated) + tambahan fields (Waktu & Modal, Run button)
- User bingung mana yang sync mana yang engga karena **secara visual berbeda**

### Goals
1. **Default koheren** ‚Äî backtest = simulation of Portfolio strategy. Pasti koheren by default.
2. **Settings UI identik** ‚Äî satu component dipakai di kedua tempat.
3. **Eksperimen tetap mungkin** ‚Äî sandbox mode eksplisit (opt-in), dengan "bridge" yang jelas untuk promote hasil ke live.
4. **Backwards compat** ‚Äî tidak break existing users (kecuali mereka manually opt-into DRAFT mode).

## Keputusan

### 1. Unified Settings Panel (NEW component)

`src/components/StrategySettingsPanel.tsx` ‚Äî reusable component yang render 10 strategy fields dalam 2 sections:

```
Section 1: Profil & Mode
  - Profil (Aman / Agresif / Dividen / Custom)
  - Mode (Algo / Custom)
  - Universe (Semua / IDX80 / IDX30 / LQ45)
  - Custom Universe (visible kalau custom mode)

Section 2: Rotasi & Risiko (collapsible)
  - Jumlah Saham (Top 1 / 3 / 5 / 10)
  - Rotasi Saham (Crossover: Rank<7 / Tanpa)
  - Proteksi Crash (ON/OFF + sensitivity slider 5-30%)
  - Safe Haven (Emas / Kas)
  - Buffer Kas (reserve, 0-30%)
```

Props:
- `config: StrategyConfigShape` ‚Äî sumber data
- `onChange(key, value)` ‚Äî write callback
- `disabled: boolean` ‚Äî saat true, semua input greyed + tooltip "Locked ke Strategi Portofolio"
- `showTitle: boolean` ‚Äî toggle title section (default true)

**Adaptive Weights** dihapus dari component ini (deprecated by ADR-010, but the toggle in Backtest sidebar was still present ‚Äî bug).

**Settings Lock toggle** + **DCA Rekomendasi** tetap di Portfolio sidebar (Portfolio-only concerns, tidak di StrategySettingsPanel).

### 2. 3 Mode + 1 Bridge untuk Backtest‚ÜîPortfolio

| Mode | Default | Description |
|------|---------|-------------|
| **Live** | ‚úÖ ON | Backtest = engineConfig. Strategy fields di sidebar greyed (disabled). Banner hijau "‚úì Live Strategy". |
| **Draft** | toggle OFF | Backtest = backtestConfig (sandbox). Strategy fields editable. Banner amber "‚öÝ DRAFT ‚Äî perubahan tidak effect Portofolio". Tombol "‚Üë PROMOTE TO PORTFOLIO" tersedia saat ada hasil. |
| **Edge case** | ‚Äì | User toggle ON dengan draft unsynced ‚Üí ConfirmModal "Buang / Promote Dulu / Batal". |

**Bridge**: `promoteDraftToEngine()` helper di EngineConfigContext copy 10 strategy fields dari `backtestConfig` ke `engineConfig`. Auto-toggle ke ON setelah promote.

### 3. EngineConfigContext changes

```ts
// New state

// New helpers
const isDraftEqualToEngine = () => /* 10 strategy keys comparison */;
const promoteDraftToEngine = () => /* copy draft ‚Üí engineConfig */;

// Removed
const isConfigSynced = ...; // replaced by isDraftEqualToEngine
```

### 4. SimulationTab changes

```ts
const effectiveConfig = backtestUseLiveStrategy
  ? { ...backtestConfig, ...engineConfig }  // engineConfig wins for 10 strategy keys
  : backtestConfig;

const result = runStrategy({
  dayData: historicalData,
  config: {
    ...effectiveConfig,  // 10 strategy keys
    capital, simStartDate, simEndDate, singleTicker  // backtest-only
  },
  ...
});
```

**10 strategy fields** yang koheren (sumber: engineConfig):
`activeProfileId`, `universe`, `topNCount`, `simulationMode`, `safeHavenAsset`, `crashSensitivity`,
`enableCrashProtection`, `customUniverse`, `enableAdaptiveWeights`, `reserveBufferPct`

**Backtest-only fields** (tetap dari backtestConfig):
`simStartDate`, `simEndDate`, `algoCapital`, `singleTicker`, `singleSellTrigger`, `singleBuyTrigger`

### 5. AppSidebar refactor

- Hapus inline `EngineConfigSidebarContent` (~200 LOC) ‚Üí pakai `<StrategySettingsPanel>` dengan `disabled={isSettingsLocked}`
- Hapus inline backtest settings (~100 LOC) ‚Üí pakai `<StrategySettingsPanel>` dengan `disabled={backtestUseLiveStrategy}`
- Tambah toggle "Gunakan Strategi Portofolio" di Backtest sidebar header
- Hapus `isConfigSynced === false` warning (sudah redundant)

### 6. PortfolioTracker UI cleanup

- 5 summary cards ‚Üí 1 hero "Net Wealth" card (Saham + Kas + Emas + P&L) + 5 mini-metrics (Modal, Nilai, P&L, Dividen/thn, Kas) dalam 1 baris
- Strategy Says banner tambah chip "IHSG live: X (drawdown Y%)" + "PROFIL: AGRESIF"
- Holdings dividen cell compact "+Rp XXjt" + "X.XX% yield"

### 7. SimulationTab metrics strip

Tambah 6-col compact metrics strip (CAGR, MaxDD, Sharpe, Dividen, Trades, Vol) di atas bento grid existing, untuk merapikan metrics yang "berantakan" karena dividen ditambahkan.

## Consequences

### Positif
- **Pasti koheren by default** ‚Äî backtest & portfolio baca `engineConfig`, hasilnya identik untuk data yang sama
- **Settings UI identik** ‚Äî 1 component, 2 tempat, 0 perbedaan visual
- **Eksperimen tetap mungkin** ‚Äî toggle OFF, edit, run, "‚Üë PROMOTE TO PORTFOLIO"
- **Edge case aman** ‚Äî toggle ON dengan draft unsynced ‚Üí modal konfirmasi (3 opsi)
- **Adaptive Weights** benar-benar dihapus (was dead code di Backtest sidebar)
- **Settings Lock + DCA Rekomendasi** tetap ada di Portfolio (Portfolio-only)

### Negatif / Trade-offs
- **"PROMOTE TO PORTFOLIO" button eksplisit** ‚Äî user harus klik untuk apply hasil eksperimen. Default behavior tidak auto-sync (was buggy: setting bisa nyangkut di draft).
- **10 fields = "banyak" kalau user eksperimen banyak** ‚Äî tapi kebanyakan user pakai default, jadi OK.
- **3 opsi di ConfirmModal** (Buang/Promote/Batal) ‚Äî sedikit kompleks, tapi lebih aman dari 2 opsi.

### Migration
- `backtestConfig` lama di `idx_engine_config` localStorage tetap di-load (tidak dihapus)
- Default `backtestUseLiveStrategy=true` ‚Üí langsung koheren tanpa intervensi user
- Existing `isConfigSynced` deleted, behavior equivalent via `isDraftEqualToEngine()`
