# ADR-009: Backtest Rank Computation Bug & Value Factor Negatif

**Status:** Accepted  
**Date:** 2026-06-25  
**Deciders:** @InitialJawa  

## Context

Saat menjalankan backtest dari 2021–2026, hasil menunjukkan return negatif (-26% Config B, -20% Config F) padahal sebelumnya dengan data Yahoo masih positif. Investigasi menemukan dua masalah:

### Bug 1: `getRanks()` di `scripts/run_backtest_comparison.cjs`

Fungsi `getRanks()` menghitung rank dari `stockNormScores` menggunakan formula:
```
ranks[t] = 1 - score
```
dimana score berkisar 40–95. Ini menghasilkan rank negatif (-94 sampai -39). Akibatnya exit condition `currentRank >= 10` tidak pernah terpenuhi, sehingga **0 swap terjadi** — portofolio buy-and-hold tanpa rebalancing.

**Fix:** Diganti dengan metode `computeDayRankings()` dari `src/engine/ranker.ts`: composite score → sort descending → rank = index + 1.

### Bug 2: Inconsistency Data `stockNormScores` vs `stockRanksProd`

- `fetch_historical_data.ts` menyimpan `stockNormScores` dengan **linear min-max normalization (40–95)** dan `stockRanksProd` dari ranking yang sama
- `migrate-normscores.ts` menimpa `stockNormScores` dengan **rank-based normalization (0–95)** tanpa mengupdate `stockRanksProd`
- Akibatnya `stockNormScores` dan `stockRanksProd` INKONSISTEN — overlap top-10 hanya 1/10 ticker

## Keputusan

1. **Value (1/PB) tidak boleh diberi bobot >5%** — ini negative-alpha factor di IDX 80 (2021–2026). Bobot 30% di Config F menyebabkan underperform.
2. **Quality (ROE) adalah faktor dominan** — bobot dinaikkan ke 40-45%.
3. **Dua profile default baru:**
   - **Config QM** (id="prod"): Q45 G10 V5 M40 — Quality Momentum, max alpha.
   - **Config BG** (id="res"): Q40 G25 V5 M30 — Balanced Growth, diversifikasi.
4. **Hasil backtest final** dengan QM:
   - QM: +150% CAGR 18.23% (vs old Config B +79%, old Config F -0.7%)
   - BG: +70% CAGR 10.21%
   - Quality Only masih tertinggi: +246% CAGR 25.5%
5. **`stockNormScores` jadi source of truth** — rank-based normalization (0-95) dari IDX warehouse.
6. **`stockRanksProd/Res` di-regenerasi** dari `stockNormScores` + profile weights — konsisten satu sama lain.
7. **Fundamental warehouse di-revert** ke versi historical (commit HEAD~1) karena collector overwrite dengan data baru yang hanya punya 13 ticker pre-2021 vs 751 ticker di versi historical.

## Consequences

- Profile "prod" (QM) dan "res" (BG) sudah diupdate dengan bobot baru
- Semua hardcoded references di UI (`AppSidebar`, `AppHeader`, `SimulationTab`, `PortfolioTracker`, `DiagnosticsTab`, `marketData`, `systemKnowledge`) sudah diupdate
- `migrate-normscores.ts` sudah diupdate dan di-run untuk regenerate data
- `run_backtest_comparison.cjs` sudah diupdate dengan weight configs baru
- Data fundamental warehouse perlu di-archive agar tidak overwrite lagi oleh collector
- Running ADR-010 untuk: archive strategi data warehouse
