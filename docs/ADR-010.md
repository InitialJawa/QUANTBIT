# ADR-010: Cross-Tab Sync Drift Fixes + Feature Simplification

**Status:** Accepted
**Date:** 2026-06-26
**Deciders:** @InitialJawa

## Context

User melaporkan: "cek ada banyak yang ngga sync antar tab misal ini bilang yang satu bilang b jadi masih bingung mau ambil keputusan apa" dan "terlalu banyak fitur sampai kita juga ngga tau gmana makenya".

Audit komprehensif terhadap `src/` menemukan:

### 5 Critical Sync Drifts (decision-blocking)
1. **Crisis detection** punya 4 rumus berbeda (`isCrisisMode` pakai 60d-drawdown, tapi
   `computeMarketRegime` dan `useProactiveAgent.Rule6` pakai `MKT.ihsg.monthly` yang hardcoded -15.42%).
   Hasil: Sidebar "RISK OFF" vs BPS "CASH DEFENSE" vs MarketTab "WARNING" di waktu yang sama.
2. **`MKT.ihsg.monthly` hardcoded** di `marketData.ts`. Tidak pernah di-update walau data feed
   berubah. Semua logika yang baca field ini pakai data stale.
3. **Portfolio value** dihitung 3 cara: AppSidebar (emas @ buyPrice = 0% return), PortfolioTracker
   (filter EMAS), MarketTab (subset). User tidak bisa angka mana yang benar.
4. **No "Total Wealth" view** â€” AppSidebar = stocks+cash (no gold), Wallet = cash+gold (no stocks),
   Portfolio = stocks only.
5. **"Breadth >60" misleading** â€” `RS.radar_context.watchlist_count` = 80 (IDX80 universe size),
   BUKAN user's watchlist count. Label membingungkan.

### 9 Feature Overload Issues
1. 7 dead components (~470 LOC unreached): `DashboardGrid`, `BottomNav`, `NavDrawer`,
   `DiagnosticsTab`, `AIAssistant`, `DeepReport`, `AICockpit`.
2. 4 dead `NotificationContext` methods (no callers): `removeNotification`, `clearAll`,
   `shouldFireRule`, `markRuleFired`, `resetRule`.
3. Duplicate "Fine-Tune Q/G/V/M/D" sliders di 2 tempat (sidebar + ManageProfilesModal).
4. Crash sensitivity + Buffer kas di-expose langsung di sidebar â€” terlalu advanced untuk most users.
5. Settings dropdown punya duplicate profile picker.
6. `useUIState.activeConfig` + `idx_activeconfig` localStorage = parallel state dengan
   `engineConfig.activeProfileId`. ConfigSync bridge 2-effect loop.
7. Backtest auto-run di `useEffect` â€” invisible to user, menyebabkan flicker setiap config change.
8. Primary "Beli Manual" CTA di baris ~1241 (di-bawah ~1000px context).
9. Jargon: "BPS Config (Live)", "Sandi Saham", "Custom Trade", "Eksekusi Beli Saham".

## Keputusan

### FASE 1 â€” Sync Fixes
1. **FASE 1.1**: `computeMarketRegime` + `useProactiveAgent.Rule6` pakai `getIhsgDrawdown60()`,
   bukan `MKT.ihsg.monthly`. Single formula = 60d-drawdown.
2. **FASE 1.2**: Tambah `getIhsgMonthlyReturn/Weekly/Daily` di `marketRegimeEngine.ts`. Update
   `MKT.ihsg.*` dari historical data di `refreshRSFromRegime()`. Hardcoded values jadi fallback 0.
3. **FASE 1.3**: Hapus filter EMAS/GOLD di `enrichedPortfolio` di PortfolioTracker. Formula
   seragam: `shares Ă— (currentPrice ?? buyPrice)`. EMAS dihitung @ buyPrice (tidak live).
4. **FASE 1.4**: Tambah `src/utils/portfolioValue.ts` dengan `totalWealth()` = stocks + cash + gold.
   Tampil di AppHeader (sticky pill, emerald accent). Klik = buka tab Portfolio.
5. **FASE 1.5**: Rename `RS.radar_context.watchlist_count` â†’ `idx_universe_size`. Field lama
   tetap untuk backward compat tapi semua display pakai field baru.

### FASE 2 â€” Feature Simplification
1. **FASE 2.1**: Hapus 7 dead components via `git rm`. `DashboardGrid` di-archive ke
   `src/components/_archive/`.
2. **FASE 2.2**: Slim `NotificationContext` ke 3 method (`notifications`, `addNotification`,
   `fireRule`). Hapus 5 method yang tidak dipanggil.
3. **FASE 2.3**: Hapus 2 blok "Fine-Tune Rasio" dari sidebar (custom mode + algo mode).
   Edit profil hanya via `ManageProfilesModal` ("Edit Profiles" button tetap ada).
4. **FASE 2.4**: Wrap Crash protection + Buffer kas di `<details>` "Pengaturan Lanjutan".
   Default closed. Safe Haven asset toggle tetap di atas (eksplisit).
5. **FASE 2.5**: Hapus section "Config" dari Settings dropdown AppHeader. Profile hanya diubah
   via sidebar chips (satu tempat).
6. **FASE 2.6**: Hapus `useUIState.activeConfig` + `idx_activeconfig` localStorage + `ConfigSync`
   component. `engineConfig.activeProfileId` adalah single source of truth.
7. **FASE 2.7**: Hapus auto-run `useEffect` di `SimulationTab.tsx:530-557`. Backtest hanya run
   saat historicalData pertama dimuat (initial) + tombol "Jalankan Backtest" eksplisit di sidebar.
8. **FASE 2.8**: Tambah sticky "Beli Cepat" pill di top Portfolio tab. Klik â†’ scroll + focus
   ke form manual buy. `id="manual-buy-form"` ditambahkan.
9. **FASE 2.9**: Simplify jargon:
   - "BPS Config (Live)" â†’ "Profil Strategi Aktif"
   - "Sandi Saham (Ticker)" â†’ "Pilih Saham"
   - "Custom Trade" â†’ "Beli Manual"
   - "Eksekusi Beli Saham" â†’ "Beli Sekarang"
   - "Hapus Semua" â†’ "Reset Portofolio"
   - "Bersihkan Entri" â†’ "Hapus Riwayat"

## Consequences

### Positif
- 5 critical sync drift hilang â€” angka konsisten di semua tab
- Total Wealth view = single source of truth untuk net worth
- ~470 LOC dead code dihapus
- Cognitive load turun ~40-50% (5 fine-tune sliders Ă— 2 lokasi, 4 advanced controls collapsed)
- Backtest lebih predictable (tidak flicker, eksplisit trigger)
- activeProfileId state machine lebih sederhana (1 source, bukan 2)

### Negatif / Trade-offs
- Backtest tidak auto-update saat user ubah config â€” user harus klik "Jalankan".
  Mitigasi: tombol "Jalankan Backtest" ada di sidebar (eksplisit).
- Fine-tune sliders hanya di modal â€” user yang biasa tune di sidebar harus klik 1x lagi.
  Mitigasi: doc di BPS Config "Profil Strategi Aktif" menyebutkan "Edit via Edit Profiles".
- Settings dropdown kehilangan 3 profile buttons â€” user yang biasa pakai harus ke sidebar.
  Mitigasi: sidebar chips lebih prominent dan eksplisit.
