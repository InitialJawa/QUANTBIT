# ADR-008: Rebalancing Engine Fix — 4 Bugs

## Status: Accepted

## Date: 2026-06-24

## Problem
Backtest engine di `src/engine/core.ts` memiliki 4 bug di blok rebalancing (line 295-350) yang menyebabkan:
1. **Day-1 false trigger** — `lastRebalanceMonth = -1` selalu ≠ current month di hari pertama → `isMonthChange = true` → sell/beli tanpa alasan di hari pertama backtest
2. **Custom mode rank-based exit** — Custom mode masuk blok rebalancing berdasar rank, padahal user memilih stock tersebut secara manual. Exit hanya boleh via crash protection atau EXIT signal
3. **Hardcoded top 4 swap candidates** — Algo mode swap mencari dari TOP 4 tickers (hardcoded), bukan `config.topNCount`. Jika user set topN=7, swap miss 3 kandidat terbaik
4. **Self-swap / duplicate position** — `swapInTicker` fallback ke `topCandidates[0]` bisa memilih stock yang baru dijual → kena fees 2x tanpa pergantian posisi. Juga bisa beli stock yang sudah dihold

## Solution

### Fix 1: `lastRebalanceMonth` init (core.ts:92)
```typescript
// Before:
let lastRebalanceMonth = -1;
// After:
let lastRebalanceMonth = new Date(day0.date).getMonth();
```
Inisialisasi ke bulan hari pertama backtest → `isMonthChange` hanya `true` saat bulan benar-benar berubah, bukan hari pertama.

### Fix 2: Custom mode excluded from rank-based exit (core.ts:295)
```typescript
// Before:
if (!inCrashState && config.enableCrossover && (config.simulationMode === "algo" || config.simulationMode === "custom")) {
// After:
if (!inCrashState && config.enableCrossover && config.simulationMode === "algo") {
```
Custom mode stock di-hold kecuali crash protection trigger. Dead code custom branch di blok swap juga dihapus.

### Fix 3: `config.topNCount` replaces hardcoded 4 (core.ts:320)
```typescript
// Before:
pickTopTickersByRank(day.stockRanks, day.stockPrices, getUniverseTickers(), 4);
// After:
pickTopTickersByRank(day.stockRanks, day.stockPrices, getUniverseTickers(), config.topNCount);
```

### Fix 4: Prevent self-swap & duplicate (core.ts:323)
```typescript
// Before:
const swapInTicker = topCandidates.find(t => !positions[t] || positions[t] === 0) || topCandidates[0];
// After:
const swapInTicker = topCandidates.find(t => t !== ticker && (!positions[t] || positions[t] === 0));
```
- `t !== ticker` — jangan beli ulang stock yang baru dijual
- Hapus `|| topCandidates[0]` fallback — jika tidak ada kandidat suitable, hold cash (bukan beli stock yang sudah dihold)

## Impact

| Bug | Mode Affected | Sebelum | Sesudah |
|-----|---------------|---------|---------|
| #1 | algo + custom | Day 1 langsung jual/beli, kena fees | Day 1 hold, rebalance mulai bulan berikutnya |
| #2 | custom | Custom stock dijual karena rank drop, lalu dibeli lagi | Custom stock di-hold, exit hanya via crash |
| #3 | algo | Swap hanya dari top 4, missed opportunity | Swap dari top N (sesuai config) |
| #4 | algo + custom | Self-swap (fees 2x), duplicate position | No self-swap, no duplicate, hold cash if no candidate |
