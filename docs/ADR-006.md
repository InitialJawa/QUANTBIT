# ADR-006: IDX API sebagai Primary Source Fundamental

## Context

QuantBit membutuhkan fundamental data (assets, equity, sales, profit, EPS,
ratios) untuk 900+ IDX stocks sejak 2015. Sebelumnya menggunakan campuran:
Yahoo Finance, FMP, Sectors.app, scraping Stockbit/RTI, dan hash fallback.

Masalah: data tidak konsisten antar sumber, cakupan terbatas, dan sulit diaudit.

## Discovery

Endpoint IDX ditemukan melalui reverse engineering Nuxt.js bundles:

```
GET /primary/DigitalStatistic/GetApiDataPaginated
  ?urlName=LINK_FINANCIAL_DATA_RATIO
  &periodYear={YYYY}&periodMonth={MM}&periodType=monthly
```

Return 32 fields termasuk assets, liabilities, equity, sales, profitPeriod,
profitAttrOwner, eps, bookValue, per, priceBV, deRatio, roa, roe, npm.

## Decision

IDX API menjadi **primary source** fundamental QuantBit.

Sumber yang di-drop:
- Yahoo Fundamental — tidak konsisten, coverage terbatas
- Financial Modeling Prep (FMP) — API key, coverage tidak lebih baik
- Sectors.app — scraping, data terbatas
- Hash Fallback Fundamental — tidak diperlukan lagi

Sumber yang ditahan (backfill only):
- RTI Business — untuk 2015-2020 jika diperlukan
- Stockbit — untuk 2015-2020 jika diperlukan

## Consequences

1. Build `collectors/fetch_idx_fundamental.py` — pull monthly 2021-01 to 2025-12
2. Simpan raw balance sheet + income statement — hitung rasio sendiri
3. Hapus dependensi Yahoo/FMP/Sectors dari pipeline fundamental
4. Perlu audit cross-check: BBCA, BBRI, BMRI, TLKM, ASII vs warehouse legacy
5. 2015-2020 deferred — 5 tahun data sudah cukup untuk V5

## Status

Accepted.

## Verification

60-month audit (2021-01 to 2025-12):
- 60/60 months available (2 transient errors pada cold start)
- 100% schema consistent (single hash across all months)
- Avg response time: 1.0s
- Rows: 711 → 960 (monotonically increasing)
- 0 rate limit issues detected
